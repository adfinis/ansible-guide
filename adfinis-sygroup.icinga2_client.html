Default values

The icinga2 parent hosts. Only 2 allowed. https://github.com/Icinga/icinga2/issues/3533

icinga2_client_monitoring_parents:
  - monitoring-master1.example.com
  - monitoring-master2.example.com

The default icinga2 parent zone

icinga2_client_parent_zone: "monitoring-master"

The API url of the icinga2 master

icinga2_client_api_url: "https://{{ icinga2_client_monitoring_parents[0] }}:5665"

The API user of the icinga2 master

icinga2_client_api_user: "root"

The API password of the icinga2 master

icinga2_client_api_pass: ""

If the client can’t reach the config master directly, set this to the next host which can reach the config master. unset by default icinga2_client_master_reachable_host: ‘’ The signing master where the cert signing will be delegated to if icinga2_client_master_reachable_host is set. icinga2_client_csr_signing_master: ‘’ Packages which should be installed additionally for custom check commands Defaults to an empty list

icinga2_client_custom_check_packages: []

List of repos to enable when installing custom check packages (rhel only) Defaults to an empty list

icinga2_client_custom_check_packages_enable_repo: []

Global Zones to be added to zones.conf

icinga2_client_global_zones:
  - global-templates

Specify if the host config & zone are managed by director and therefore should not be written by the master playbook

icinga2_client_director_managed: false

Specify whether the client should actively try to connect (false) to its parent(s) or not (true)

icinga2_client_passive: false
